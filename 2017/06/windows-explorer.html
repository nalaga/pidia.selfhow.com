Windows Explorer

Windows Explorer

Windows Explorer(윈도우즈 익스플로러)란, Windows 95로 Windows NT 4.0으로부터의 Microsoft Windows에 대하고, 파일 시스템에 액세스 하기 위해서 사용되는 객체 지향형의 그래피컬 유저 인터페이스이다.Windows 3. x까지의 파일 관리자와 프로그램 매니져를 옮겨놓은 것으로, 단지 익스플로러, Explorer와 표기되는 것도 많다.

목차

개요

일반적으로 Windows XP이전의 유저에게 있어서 익스플로러라고 하면 폴더 트리가 좌측으로 표시된 폴더의 표시된 윈도우가 표시되는 Windows의 파일 관리 기능을 가리키는 것이 많다.그 윈도우의 타이틀이 「(폴더명) -익스플로러」가 되고 있는 것이나, 스타트 메뉴안에 그것을 기동시키는 「익스플로러」라고 하는 항목이 존재해, 그것은 Explorer.exe에의 쇼트 컷이 되어 있기 때문에 있다.

그러나 실제의 익스플로러는 그 뿐만 아니라, 통상의 폴더 표시, 태스크바와 스타트 메뉴, 데스크탑에 표시되는 아이콘 등 Windows의 쉘로서 여러가지 기능을 담당하고 있다.

익스플로러의 윈도우는 복수의 페인(소창)에 분할되어 통상은 좌단에 표시되는 페인은 Windows XP이전으로는 익스플로러 바로 불린다.폴더의 트리 구조외, 파일 검색 기능, Internet Explorer의 이력이나 마음에 드는 것을 표시할 수도 있다.

Windows Explorer와 Internet Explorer

Internet Explorer 버전 4.0이후를 통합 인스톨 한 Windows 95로 Windows 98이후의 버전으로부터, Internet Explorer 6을 탑재한 Windows XP까지는 Windows Explorer와 Internet Explorer가 똑같이 OLE을 사용하므로 같은 조작을 실시할 수 있게 되어 있다.구체적으로는, Windows Explorer의 폴더 표시의 윈도우로 URL을 입력하면 WWW나 FTP에 액세스 할 수 있거나 반대로 Internet Explorer로 자신의 컴퓨터의 폴더에의 패스를 지정해 그 폴더를 표시시킬 수도 있는 것이다.

이것으로부터 양자는 동일한 프로그램이다고 생각되는 일이 있지만, 양자는 여전히 다른 프로그램이다.예를 들어 태스크바는 Windows Explorer의 기능이기 위해, 어떠한 원인으로 Windows Explorer만 이 종료되어 태스크바가 사라졌을 때, Internet Explorer를 기동해도 태스크바는 표시되지 않고, Windows Explorer를 기동시키지 않으면 태스크바는 표시되지 않는다.이것으로부터 Windows Explorer와 Internet Explorer는 전혀 같은 것은 아닌 것을 알 수 있다.

Internet Explorer 7이후로는 이 기능은 삭제되어 Windows Explorer로 URL을 입력하면 표준의 웹 브라우저(Internet Explorer라고는 할 수 없다)의 윈도우가 새롭게 열어, Internet Explorer로 폴더에의 패스를 지정하면 Windows Explorer의 윈도우가 새롭게 열게 되었다.

액티브 데스크톱

Windows 95이후에 Internet Explorer 버전 4에서 6을 도입하는 것으로써 액티브 데스크톱이라고 하는 기능이 익스플로러에 통합된다.IE4~6이 표준 탑재된 Windows 98~XP는 액티브 데스크톱도 표준 탑재되고 있다.태스크바내에 툴바(퀵 기동, 데스크탑 등 특정의 폴더의 내용이 표시된다)를 추가할 수 있거나 데스크탑에 임의의 Web 페이지를 붙이거나 Windows bitmap 형식 이외의 화상(JPEG등 )이나 HTML 문서를 벽지로서 사용할 수 있는 등의 기능이 있다.또 익스플로러에 IE의 「마음에 드는 것」메뉴가 표시되어 익스플로러 바에 IE의 마음에 드는 것이나 이력을 표시할 수도 있다.

데스크탑과 Web의 통합이라고 하는 개념은 웹 브라우저의 부당한 쉐어 독점과의 균형으로 비판의 대상이 되었다.또 시큐러티면의 염려도 있어, 퀵 기동 툴바등을 제외해 많은 지지를 받았다고는 말하기 어렵다.

Windows Vista 이후로는 액티브 데스크톱은 삭제되었다.익스플로러에는 IE와는 별도로 관리되는 마음에 드는 것이 도입되었다(폴더에의 쇼트 컷을 추가할 수 있지만, Web 페이지의 URL은 추가할 수 없다).데스크탑의 Web 항목은 가제트에 역할을 양보해, 너무 유효하게 활용되지 않았던 전자와는 달라 큰폭으로 기능성이 향상했다.

특징

Windows 95로 Windows NT 4.0

Windows Explorer 최초의 버전이다.프로그램 매니져와 파일 관리자를 통합했다.프로그램 매니져의 기능은 스타트 메뉴에, 파일 관리자의 기능은 데스크탑에 짜넣어져 종래의 프로그램과의 호환성을 가진다.NeXTSTEP의 쉘을 거의 모방한 디자인, Mac OS의 Finder와 같은 쓰기를 갖춘다.빌 게이츠와 스티브・죠브즈의 사이가 좋고, 또 게이츠가 죠브즈의 실적을 높게 평가하고 있던 고로 실현되었다. Windows NT 4.0의 Windows Explorer는 Windows 95판과 동일한 것에, 시큐러티 설정 기능을 추가한 것.복수의 유저가 스타트 메뉴를 공유할 수 있도록, 표준 상태, 전유저의 설정 상태, 개별 유저 마다의 설정을 보관 유지 할 수 있는 것처럼 변경.

Windows 98

Windows 네트워크 브라우저 기능이 추가되었다.Windows CE가 갖추는 태스크바로 윈도우의 최소화・복원의 기능이 반영되었다.디렉토리의 링크를 전개해 표시 할 수 있는 것처럼 되었다.

Microsoft Plus! for Windows 98을 인스톨 하면 압축 폴더 기능(익스플로러로 ZIP의 열람・해동이나 부분 해동・압축을 할 수 있다)이 추가된다.Windows 2000이후로는 압축 폴더는 표준 탑재되고 있다.또한 Plus!(을)를 도입하지 않아도 CAB 형식의 열람・해동은 가능.

Windows 2000으로 Windows Me

검색범위의 추가.Windows 2000의 Windows Explorer만, 비디오 파일이나 사운드 파일의 프리뷰를 위한 미디어 player가 짜넣어지고 있다.

Windows XP와 Windows Server 2003

WindowsXP로부터 스타트 메뉴가 확장되어 종장 2열의 큰 메뉴가 열리게 되었다.종래 데스크탑에 배치되어 있던 마이 다큐먼트나 내 컴퓨터등의 폴더가 항목으로서 표시되는 것 외에 과거에 이용하고 있던 어플리케이션을 리스트업 하는 기능이 추가되어 이것들이 표준이 되었다.일부의 항목을 리스트 항목으로 변경하거나 비표시로 할 수 있는 것 외에 스타트 메뉴 자체를 종래의 1열의 디자인에 되돌리는 일도 할 수 있다.Luna외, XP로부터 도입된 UI의 스킨 기능에 완전 대응하고 있다. Windows Server 2003은 Windows XP와 동일한 것이지만, 검색 컴패니언과 테마의 이용을 표준으로 무효로 한 것.

태스크 패널
파일이나 폴더의 변경이나 빈번히 참조되는 곳에의 링크나, 파일의 상세를 표시하는 영역이 추가되었다.
검색 컴패니언
파일 검색을 실시하는 영역에 컴패니언을 표시하게 되었다.이 컴패니언은 Microsoft Office에 있던 컴패니언의 외, 새롭게 추가된 것도 있다.
화상 표시
화상 표시는 2종류있어, 파일 아이콘에 직접 표시하는 것과 폴더 아이콘에 직접 표시하는 것이 추가되었다.폴더 아이콘은 아이콘의 사이즈를 크게 했을 경우에게만 처리된다.
Web 발행
Web 호스팅을 실시하는 서비스 프로바이더에 파일을 업 로드하기 위한 기능.FTP나 HTTP를 이용해 Web 발행 위저드를 사용해 임의의 파일을 올바르게 업 로드한다.
그 외의 변경
Windows Explorer상에서 CD 또는 DVD에의 파일의 기입을 실시하는 기능이 추가되었다.

Windows Vista와 Windows Server 2008

Windows Aero의 도입에 수반해 인터페이스가 큰폭으로 변경되었다.

검색
Explorer의 상부에 검색 박스를 설치했다.또, 이전부터 있던 검색 기능에 대해서도, 검색 컴패니언을 중지하고, 검색 기능에 특화하듯이 새롭게 설계되었다.메타데이타 검색과 풀텍스트 검색의 양쪽 모두를 실시하도록(듯이)도 변경되었다.
레이아웃의 변경
  • 툴바에는 빈번히 사용하기 위한 버튼만을 표시해, 도구모음을 표준으로 표시하지 않게 변경되었다.또 XP이전으로는 버튼의 추가・삭제나 레이아웃의 변경이 가능했던 툴바의 커스터마이즈가 불가능이 되었다.
  • 주소 바는 폴더명 또는 풀 패스를 표시할 만한 종래의 것에 대신해, 빵 쓰레기 리스트로 불리는 폴더등의 트리 구조를 더듬기 쉬운 표시 형식이 도입되었다.
  • 직하의 주소 바에 현재의 패스가 표시되기 위해, 타이틀 바에는 아무것도 표시되지 않게 되었다.(다만, 클래식 스타일 적용시는 종래대로 폴더명이 표시된다.)
  • 윈도우 하부의 상세 페인은 종래의 상태 바에 대신하는 것으로, 보다 많은 파일 정보를 표시할 수 있는 것 외에 JPEG 파일・Microsoft Office 문서등의 상세 정보나 MP3 파일의 ID3 태그등을 프롭퍼티를 열지 않고 편집할 수 있다.또한 종래의 상태 바도 남아 있다.
  • 나비게이션페인… XP이전으로는 익스플로러 바로 불리고 있던 윈도우 좌측의 영역에는, 마음에 드는 것(잘 사용하는 폴더에의 링크)과 현재 위치까지의 폴더 트리가 상하에 줄지어 있다.
아이콘
아이콘의 사이즈를 이전보다 큰 표시에 대응하게 되었다.디폴트의 아이콘 사이즈도 XP이전보다 커졌다.
폴더의 종류
폴더에 포함되는 파일의 종류에 의해서 「뮤직」 「비디오」 「픽쳐」등이 자동적으로 설정되어 그것들에 최적화한 표시 형식으로 변경된다.예를 들면 비디오나 픽쳐의 경우는 큰 엄지손가락 프리뷰가 표시된다.
폴더 마다 종류를 커스터마이즈 하는 것은 가능하지만, 변경의 보존수에 상한이 있기 위해 설정이 낡은 분으로부터 없어져 익숙해진 표시 형식이 돌연 바뀌어 버리는 현상이 종종 발생한다.레지스트리의 변경에 의해 무효화해, 모든 폴더의 표시 형식을 통일할 수도 있다.
컨트롤 패널의 통합
컨트롤 패널내의 여러가지 설정 항목이 익스플로러의 단일 윈도우내에서 조작할 수 있게 되었다.다만 상세한 설정은 종래대로 다이알로그 박스를 열어 실시한다.Microsoft Update도 Internet Explorer는 아니고 익스플로러내에서 조작이 완결하게 되었다.
파일이 관련지어 관리의 변경
파일 형식이 관련지어를 관리하는 기능이 XP이전의 폴더 옵션으로부터 컨트롤 패널의 「기정의 프로그램」에 옮겨졌다.디폴트로 열리는 이외의 액션이나 파일 아이콘의 변경이 불가능하게 되어, 그것들을 변경하려면 레지스트리를 직접 편집할까 서드 파티제의 소프트웨어가 필요했다.
복수 기동의 억제
예를 들면 데스크탑에 있는 폴더의 쇼트 컷을 더블 클릭 계속 하면, XP이전으로는 같은 폴더를 표시하는 익스플로러의 윈도우를 몇 개에서도 열 수 있지만, Vista로는 이미 열려 있는 윈도우를 맨 앞면으로 하는 것이 될 뿐이어, 같은 내용의 윈도우가 복수 일어서는 것을 막게 되었다.
그 외의 변경
「내 컴퓨터」의 아이콘이 「컴퓨터」로 개칭되었다.
Vista SP2로부터 Blu-ray Disc (BD-R・BD-RE)의 써에 대응했다.

Windows 7으로 Windows Server 2008 R2

명칭의 변경
2008년 7월 25일에 마이크로소프트 일본 법인이 발표한 새로운 장음의 표기 기준[1]에 근거해, 일본어판으로는 「익스플로러」라고 표기가 변경되었다.관련하는 용어도 「컴퓨터」 「폴더」 등 장음이 부가되었다.
라이브러리
복수의 장소에 있는 미디어 파일이나 문서를 라이브러리에 추가하면 일원적으로 관리할 수 있다.디폴트로는 익스플로러를 열면 라이브러리가 표시된다.Vista에의 탑재를 단념한 WinFS의 기능을 한정적으로 실현된 형태.
나비게이션페인
Vista의 나비게이션페인을 개량해, 마음에 드는 것・라이브러리・홈 그룹・폴더 트리・네트워크가 정리해 표시된다.
데스크툽가제트
Windows 사이드바와 통합해, 가제트는 처음부터 데스크탑의 임의의 위치에 추가할 수 있다.
그 외의 변경
파일의 프롭퍼티나 상세 페인으로의 정보 표시가 AAC나 DivX등이 새로운 형식에 대응.
LHA 형식의 압축파일을 표시・해동 가능하게 되었다.
파일의 갱신 시각등의 초의 표시를 할 수 없게 되었다.다만 파일 시스템 자체에는 초이하도 기록되고 있으므로, 써드파티제 소프트등을 사용하는 것에 의해서 알 수도 있다.

Windows 8

레이아웃의 변경
최대의 변경점은, Microsoft Office 2010의 어플리케이션과 같게 커멘드 버튼은 윈도우 상부의 리본에 정리하고 있다.탭은 아무것도 하지 않을 때가 2개, 선택하는 파일에 의해서 바뀐다.
스타트 메뉴의 폐지
화면 좌하의 스타트 메뉴가 없어져, 스타트 화면으로부터 데스크탑을 선택하는 일로 Windows Explorer에 액세스 할 수 있게 되었다.스타트 화면은 전화면을 점유 해, 또 어플리의 종류에 따라서는 스크롤 시키지 않으면 안 되는 등 , 전세계로부터 많이 악평을 샀다.
God Mode의 추가
신규 폴더를 작성하고, GodMode. {ED7BA470-8 E54-465 E-825 C-99712043 E01C}라는 이름으로 변경하면, 모던 어플리나 프롭퍼티, 컨트롤 패널에 산란해 버린 시스템 설정을 일괄해 액세스 할 수 있는 특수한 폴더를 만들 수 있다.이것은 Windows 10에서도 계승되고 있다.

Windows 8.1

명칭의 변경
Windows 7이나 Windows 8으로는 「컴퓨터」라고 표시되고 있던 장소가, 일본어판은 「PC」가 되어, 영문판은 「This PC」가 되었다.
시작 버튼의 부활
시작 버튼을 누르면, 스타트 화면이 불려 가는 것처럼 되었다.이것도 또 전세계로부터 「이것은 다르다」라고 악평을 샀다.

Windows 10

Build 10240

코드네임 Threshold.마지막 테크니컬 프리뷰・최초의 공식 공개 버전.

스타트 메뉴의 부활
Windows 7의 스타트 메뉴와 Windows 8의 스타트 화면의 절충을 취한 디자인이 되었다.시작 버튼을 누르면, 왼쪽 반은 Windows7 라이크의 물건이, 오른쪽 반은 Windows8의 스타트 화면의 양쪽 모두가 표시되어 전화면을 점유 하지 않게 되었다.종래 Windows 7의 메뉴 우측에 있던 퀵 스타트 항목은 익스플로러의 각 윈도우에 개별적으로 퀵 엑세스로서 표시된다.Microsoft는 공식 공개와 함께 종래의 쓰기가 부활했다고 선언했다.
커스터마이즈
스타트 메뉴나 익스플로러 윈도우의 거동은 Windows 7에까지 거슬러 올라가 기능이나 레이아웃의 커스터마이즈를 할 수 있다.또 사이즈가 가변이 되어, 정보가 표시를 다 할 수 없게 되면 스크롤 바가 나타난다.타블렛 모드로 하는 일로 손가락으로 조작하기 쉬운 Windows 8의 스타트 화면으로 하는 일도 할 수 있다.

Build 10586

코드네임 Threshold 2

Cortana의 완전 실장
10240으로는 영구권만의 서포트인 Cortana가 국제화 되었다.음성인식으로 Winows 내외에의 정보로 액세스 할 수 있다.특히 키보드를 갖추지 않는 디바이스로 Windows의 오브젝트에의 액세스가 현격히 용이하게 되었다.예를 들면 컨트롤 패널을 열 때, 마이크 아이콘을 탭 해 「컨트롤 패널」이라고 말하면, 컨트롤 패널에의 링크가 제시된다.

RedStone(개발중) 편

VR기술의 통합
데스크 탑 환경을 망막 투영 VR디바이스에 의해, 현실의 공간과 통합한, 새로운 익스플로러가 공개되었다.

확장성

Windows Explorer는, 「Windows 쉘 확장」에 의해서 표준으로는 가지고 있지 않은 기능을 추가할 수 있다.쉘 확장은, 익스플로러에 대한 플러그 인으로서 기능하는 COM 오브젝트인[2].

쉘 확장에는, 다음과 같은 형태가 존재한다: 프리뷰 핸들러, context menu 아이템, 툴바, 쉘 이름 공간 확장.덧붙여 이름 공간 확장이란, 특정의 폴더(파일 시스템상의 존재에 한정되지 않는, 예를 들어 스캐너로 수중에 넣은 화상등)를 특수 폴더로서 취급하도록(듯이) 하는 기능이다.게다가 익스플로러에는, 메타데이타를 파일 본체와는 별도로 NTFS의 대체 데이터 스트림에 보존하는 기능이 존재하는[3].

프리뷰 핸들러는 파일로부터 엄지손가락의 전개를 실시한다.이 엄지손가락은, 파일이 선택되거나 삼네이르뷰가 액티브하게 되었을 때에 표시된다.프리뷰 핸들러는 파일 타입마다 실시할 필요가 있다.context menu 아이템은 특정의 파일의 종류, 또는 전파일을 대상으로, 어느 파일을 선택했을 때의 메뉴(도구모음의 파일 또는 오른쪽 클릭등에 의한 context menu)에 항목을 추가하는 확장이다.Vista로부터 쉘 확장의 종류에 프롭퍼티 핸들러가 추가되었다.이것은 특정의 파일의 종류로, 파일의 메타데이타를 추출하는 것으로, 상세 표시의 항목에 더해지는[4].

쉘 이름 공간 확장은, 익스플로러상에서 폴더님이 데이터를 표시시키거나(취급하는 대상이 실재하는 파일일 필요는 없다), 혹은 실재하는 파일의 구성과는 다른 표시를 실시하거나하기 위한 쉘 확장이다.「내 컴퓨터」나 「네트워크 플레이스」와 같은 특수 폴더는, Windows 익스플로러 자신이 쉘 이름 공간 확장을 실장한 것이다.예를 들면, 버전 콘트롤 시스템으로는, 익스플로러로 각 리버젼을 열람할 수 있듯이 하기 위해서 이름 공간 확장을 이용하고 있는 것도 있다.이름 공간 확장을 제공하려면 , IPersistFolder, IShellView, IShellFolder, IShellBrowser, IOleWindow등이라고 하는 COM 인터페이스를 실장할 필요가 있다.이것들은, 표시 시키려고 하는 데이터에 대해서, 폴더로서의 조작의 논리를 제공하기 위해서 필요하게 된다.그리고, Windows 익스플로러는 필요에 따라서 이것들을 실장한 COM 오브젝트를 실체화(인스턴스를 생성)하는[4].

Windows 익스플로러는 COM 인터페이스를 확장을 실시하는 기점으로서 취급하고 있기 때문에,. NET Framework에서도 COM Interop을 이용해 쉘 확장의 작성이 가능한[4].마이크로소프트 자신도 「Photo Info tool」와 같은. NET Framework상에서 동작하는 쉘 확장을 작성한[5].그럼에도 불구하고, 현재,. NET Framework를 이용한 쉘 확장의 작성은 추천 되어 있지 않다.이것은, 1개의 프로세스내에서 실행할 수 있는 CLR는 1개의 버전만이라고 하는 제한으로 연관된다.동시에 읽히는 복수의 쉘 확장이 다른 버전의 CLR를 이용하는, CLR 버전의 충돌이 일으켜져 버리기 때문에 있는[6][7].
